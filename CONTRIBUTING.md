# Contributing to Swift iOS Skills

Thank you for your interest in contributing! This repository thrives through community feedback, improvements, and shared learnings.

---

## 🤝 How to Contribute

### Report Issues
Found a bug, typo, or outdated pattern? [Open an issue](https://github.com/marinodev/swift-ios-skills/issues)

```
Title: "[Skill Name] Issue description"
Description: 
- What's the problem?
- What iOS version/Swift version?
- Example code if applicable
```

### Suggest Improvements
Have a better way to implement a pattern? Share your idea:

```
[Discussions](https://github.com/marinodev/swift-ios-skills/discussions)
→ New discussion → Suggest improvements
```

### Submit a Pull Request

1. Fork the repository
2. Create a branch: `git checkout -b improve/skill-name`
3. Make your changes
4. Test locally
5. Submit PR with description of changes

### Improve Existing Skills

Each skill can be enhanced with:
- Better error handling patterns
- Performance optimizations
- Additional real-world examples
- Clearer explanations
- Better testing strategies

### Add Your Use Cases

Have you used these patterns successfully? Share your story:

```
Discussions → Share your projects/learnings
```

## 📋 Contribution Guidelines

### Code Quality

- [ ] Code tested and working
- [ ] Follows existing patterns in the skill
- [ ] Error handling included
- [ ] Comments explain "why", not "what"
- [ ] Checklist updated (if applicable)

### Documentation

- [ ] Clear explanation
- [ ] Real-world example included
- [ ] Links to related skills
- [ ] iOS version compatibility noted
- [ ] Common mistakes section (if applicable)

### Skills Requirements

When creating a new skill:

- [ ] Production-ready code (not examples)
- [ ] 300-500 lines with examples
- [ ] iOS 17+ only
- [ ] No external dependencies (core skills)
- [ ] Swift Testing framework
- [ ] MIT license compatible
- [ ] Real-world use case documented
- [ ] Implementation checklist included
