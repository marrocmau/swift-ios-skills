## 🎯 Why This Repository?

In the era of **Apple Intelligence**, building an iOS app requires more than just code; it requires **Intent-First Design** and **High-Performance Foundations**. This repo provides:

- ✅ **AI-Native Patterns:** Orchestration of system Foundation Models and LLMs.
- ✅ **Intent-First Architecture:** App Intents as the semantic "brain" for Siri.
- ✅ **Liquid Glass Design:** Organic, reactive UI standards for 2026.
- ✅ **High-Performance Core:** Low-latency Swift (Span, InlineArray) for on-device AI.
- ✅ **System-Wide Reasoning:** Making your app a seamless plugin for Apple Intelligence.
- ✅ **Zero Dependencies:** Pure Swift, native frameworks, full control.

## 📦 The 5-Phase Roadmap

### 🔴 Phase 1: Foundation & Core (AI-Ready)
*Core implementations optimized for data-heavy AI synchronization.*
- **ios_mr_SwiftUIArchitecture.md:** MVVM + @Observable, dependency injection.
- **ios_mr_Networking.md:** URLSession + **AI Streaming (AsyncSequence)**.
- **ios_mr_SwiftData.md:** Models, Inheritance & **Batch Background Sync**.
- **ios_mr_HighPerformanceSwift.md:** **Span, InlineArray & Ownership** (NEW).
- **ios_mr_Testing.md:** Swift Testing @Suite & @Test patterns.
- **ios_mr_Security.md:** Keychain, Face ID, and Privacy Manifests.

### 🟡 Phase 2: Automation & Scale (Intelligence-Ready)
*Monitoring, monetization, and global reach.*
- **ios_mr_PushNotifications.md:** Optimized for **AI Summaries** and Relevance.
- **ios_mr_Localization.md:** **String Catalogs (.xcstrings)** and Dynamic AI Translation.
- **ios_mr_Analytics.md:** Event Tracking + **MetricKit Performance Telemetry**.
- **ios_mr_Revenuecat.md:** Subscriptions and In-App Purchases.
- **ios_mr_Fastlane.md:** CI/CD and automation.
- **skill-sentry-setup.md:** Error tracking and crash reporting.

### 🟢 Phase 3: UI Polish & System Integration (Sensory Design)
*App presence that feels "alive" on the device.*
- **ios_mr_Animations.md:** **Liquid Glass Basics**, PhaseAnimators, and MeshGradients.
- **ios_mr_LiveActivities.md:** Predictive updates via **App Intents** and Dynamic Island.
- **ios_mr_Widgets.md:** Home & Lock screen with **TimelineEntryRelevance**.
- **ios_mr_DeepLinking.md:** Navigation and Universal Links.
- **ios_mr_ASO.md:** App Store Optimization and Metadata.

### 🔵 Phase 4: Advanced Features (The "Brain")
*Deep integration with Apple Intelligence system capabilities.*
- **ios_mr_OnDeviceAI.md:** Orchestration of **Foundation Models** (Local Reasoning).
- **ios_mr_AppIntents.md:** **Assistant-Integrated Schema** for Siri.
- **ios_mr_Accessibility.md:** **Semantic UI** for Visual Intelligence navigation.
- **ios_mr_StoreKit.md:** Advanced StoreKit 2 monetization.

### 🟣 Phase 5: Future-Proofing (Xcode 26 Ready)
*The absolute cutting edge of iOS development.*
- **ios_mr_LiquidGlass.md:** Organic, Reactive UI & MeshGradients.
- **ios_mr_IntelligenceOrchestration.md:** Agent-centric multi-turn reasoning.
- **ios_mr_VisualIntelligence.md:** Camera-first contextual actions.

## 🚀 Quick Start

### 1. Clone & Browse
```bash
git clone https://github.com/yourusername/swift-ios-skills-repo.git
cd swift-ios-skills-repo
```

### 2. Implementation Guide
1. Start with **Phase 1** to build your foundation.
2. Read `phase1/ios_mr_HighPerformanceSwift.md` to optimize for local AI.
3. Use **Phase 4 & 5** to expose your app to Apple Intelligence.
4. Reference `docs/INDEX.md` for a complete file-by-file map.

### 3. Use in Claude Code / Cursor
```
Claude: "Read phase4/ios_mr_AppIntents.md and phase5/ios_mr_IntelligenceOrchestration.md. 
Implement a semantic intent that Siri can use to summarize my active tasks."
```
